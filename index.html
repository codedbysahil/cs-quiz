Architectural planning

SYSTEM BLUEPRINT

The application uses a tokenized CSS foundation, a strict structural grid, and one in-memory state object. Rendering is centralized so filters, score changes, mode switches, and flip animation all resolve through the same deterministic update loop.

01 / CSS variable architecture

Token layers

Colors, spacing, type scale, motion curves, radii, shadows, and cursor coordinates all live at root scope, making the interface easy to reshape without touching component internals.

  • One accent color reserved for hover, badges, and signal states.
  • Oversized display type paired with compact mono metadata.
  • Shared easing language for reveal, flip, and navigation.
02 / Layout grid

Structured chassis

The page is divided by full-width 1px lines, then organized into a hero, a planning band, and a split workspace with a utility sidebar and a cinematic flashcard stage.

  • Sticky header keeps high-value controls permanently available.
  • Blueprint borders define every zone with technical precision.
  • Container-aware card typography holds shape on smaller widths.
03 / Vanilla JS state machine

Event core

The JS layer stores active topic, search term, deck mode, filtered deck, current index, flip state, score map, and streak value. Every interaction mutates state, then re-renders the interface from that source of truth.

  • Topic and search filters rebuild the working deck.
  • Keyboard shortcuts mirror every primary action.
  • No framework, storage, or external interaction library.
